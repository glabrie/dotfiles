* About

This is a literate config for Emacs, my editor of choice. I suggest not taking it wholesale unless you have my exact setup. I have mixed declarative NixOS with mutable Emacs, so this configuration file has in it the nix file needed to declare it.

Also note that this being a literate config, it needs to be tangled for it to create the config files themselves. You can do so inside Emacs with ~C-c C-v C-t~, or ~M-x~ to get the function ~org-babel-tangle~.

Special thanks to [[https://github.com/SophieBosio][Sophie]], your configuration was a joy to explore and much of the final result here is lifted of inspired by what you did.

I'm mainly a hobbyist. I like diving into new things and learning.
 [[https://www.gnu.org/software/emacs/][Emacs ]]is a program released in 1985, 41 years ago. It's a bit older than me, by some months. Being from another time but still actively developed and maintained to this day, Emacs evolved into its own beast, the product of a divergent evolution with its very own ethos. Growing in isolation from UI/UX trends and standards over these four decades, Emacs forged its own path. A very different architecture creates a nomenclature so alien it sometimes feels obscure, almost occult in a way. There is definitely something magical in there.

In 40 years, I've changed so much that it might be hard to chart me as a single individual line. Not so with Emacs. Its roots are still there and vibrant, sometimes for the best, sometimes introducing quirks you'd never see elsewhere. It's part of the charm of it.

Emacs sells itself as a text editor, but I would argue (like many) that it's really an Emacs lisp interpreter that went haywire trying to recreate an OS. That interpreter affords us the magic that Emacs summons as it becomes this mutable platform that blurs the line between writing in an application and writing the application itself.

Now, that Lisp machine can do anything your mind conjures. Years of hacks, packages, extensions are at your fingertips.

** Org-mode and Org-roam

#+begin_quote orgmode.org
A GNU Emacs major mode for keeping notes, authoring documents, computational notebooks, literate programming, maintaining to-do lists, planning projects, and more — in a fast and effective plain text system.
#+end_quote

Org is a plain text file format not unlike markdown with a variety of awesome features. Org-mode is a major mode of Emacs taking advantage of that format.

It has various functions for tables (that can act as spreadsheets, much more than simple tables in markdown), has export and publish options and even a clocking function, to track time. It's also highly extensible, with numerous packages to add to Org-mode.

It's that big, while still being at it's heart just a plain text file.

*** An example of the format

#+begin_src org
,#+title: An example of an org format file
,#+author: Ghil
,#+date: <2026-05-14 Thu>

Metadata about the file are found up top, with a #+ prefix. There are also options for specifying the language, an html meta description and export options, like tangling, which I'll explain in the literate programming section.

,* Headers use asterisks
,** And can be nested
,*** infinitely.

But headers are more than just titles. They can be tasks as well, tracked across all your org files with the agenda function inside Emacs (Evil-mode: SPC o a , vanilla: C-c a ). You can set yourself a TODO, you can either schedule it or put a deadline on it, even place some sub-tasks in it and get a progress percentage automatically on that header.

,* TODO Finish writing NixOS literate config [44%]
,SCHEDULED: <2044-06-01 Mon>
#+end_src

*** Literate programming

Another thing that sets org apart is the concept of literate programming. It basically is a way to wrap your code inside a prose document. The advantages are two fold: Documentation is embedded with the code, giving you the coveted *One Source of Truth* ™ for your project. It also allows for more flexibility when commenting the code, giving you access to a large array of formatting options just not found in // blocks in code.

This file is a literate document, which starts has a .org . In it, we can use SRC blocks to put our code in, like so:

#+begin_example
,#+begin_src
Your brilliant code here!
,#+enc_src
#+end_example

** Opacity problems
:PROPERTIES:
:CUSTOM_ID: issue-opacity
:END:

- Last investigated :: [2026-05-04]
- Severity :: Cosmetic

*** Description of the issue

You can now apply transparency to emacs's background natively with this bit:

#+begin_src emacs-lisp
(set-frame-parameter nil 'alpha-background 70)
(add-to-list 'default-frame-alist '(alpha-background . 70))
#+end_src

Note that you need a PGTK package of emacs for it to work on Wayland. With that done, you should have a nice transparency effect on the background only of emacs' frames. The issue I am having with it is in regard to the leading asterisks behind the header levels.
Normally, headers are denoted with stars. The depth is denoted by the number of stars, like so:

#+begin_example
,* Level 1
,** Level 2
,*** Level 3
#+end_example

Org-mode will normally attempt to hide the stars by using the ~Org-hide~ [[*Faces][face]], and then use ~org-indent-mode~ to indent them visually.
Concretely, when ~org-hide-leading-stars~ is set to ~t~, a font-lock keyword is installed that will apply ~org-hide~ to the stars. That face has a ~:foreground~ color property that ~Org~ will set to match the frame's ~:background~ color property, making them appear invisible. This would normally be fine, except when transparency is turned on. Now that background is not static, and your background has an opacity value, the hidden stars shine through at 100% opacity, leaving faint black marks behind the bullet that you want shown, like here:

[[../../../assets/issue-opacity-1.png]]

*** Attempted fixes and knowledge gained.

+ I initially tried to play with ~org-modern-hide-stars~, setting it to ~nil~ or to or to ~" "~.
+ Disabled ~org-startup-indented~ to diagnose. It did remove the problem at the cost of visual indenting.

**** Incidental knowledge

+ ~org-modern-hide-stars " "~ is a clean alternative that bypasses the face mechanism entirely but for some reason seem to disable itself when ~org-indent-mode~ is active, as far as I could see.
+ ~org-indent-mode~ which is enabled by ~org-startup-indented t~, seems to reset any global changes made by ~org-modern-hide-stars t~ at the local buffer level, preventing it from firing at all.

*** Resolution

As of [2024-05-04] I chose to remove transparency from emacs. I will revisit this someday.

*** Emacs as a server

One of the understated advantages of emacs is being able to run it as a server and have new frames attach to it automatically. This has several benefits here:
- Doom Emacs is started just the once, meaning that actually opening an emacs frame (the app) is, wait for it, *Blazingly Fast*.
- Since the daemon is running in the background, processes like mu4e or the killring (copy-paste) don't have to restart between frames, you just reattach each time. It also means you can have several frames opened that link to the same buffers (Let's say you have one instance and another in the terminal for example).
- I also use it as a global org-mode capture with a hotkey in niri ( ~Mod+C~ ) that make noting things on the fly a breeeze.

We'll also start mbsync, which is necessary for mu4e.

#+begin_src nix
    services = {
      emacs.enable = true;
      mbsync.enable = true;
    };
#+end_src

                (svg-lib-progress-bar (/ (string-to-number value) 100.0)
                  nil :margin 0 :stroke 2 :radius 3 :padding 2 :width 11)
                (svg-lib-tag (concat value "%")
                  nil :stroke 0 :margin 0)) :ascent 'center))

  (defun svg-progress-count (value)
    (let* ((seq (mapcar #'string-to-number (split-string value "/")))
           (count (float (car seq)))
           (total (float (cadr seq))))
      (svg-image (svg-lib-concat
                  (svg-lib-progress-bar (/ count total) nil
                    :margin 0 :stroke 2 :radius 3 :padding 2 :width 11)
                  (svg-lib-tag value nil
                    :stroke 0 :margin 0)) :ascent 'center)))

  (setq svg-tag-tags
        `(;; Task priority
          ("\\[#[A-Z]\\]" . ((lambda (tag)

                                             :beg 2 :end -1 :margin 0))))
          ;; Progress

            (svg-progress-percent (substring tag 1 -2)))))
          ("\\(\\[[0-9]+/[0-9]+\\]\\)" . ((lambda (tag)
            (svg-progress-count (substring tag 1 -1)))))
          ;; Active date
          (,(format "\\(<%s>\\)" date-re) .
           ((lambda (tag) (svg-tag-make tag :beg 1 :end -1 :margin 0))))
          (,(format "\\(<%s \\)%s>" date-re day-time-re) .

          (,(format "<%s \\(%s>\\)" date-re day-time-re) .
           ((lambda (tag) (svg-tag-make tag :end -1 :inverse t :crop-left t :margin 0))))
          ;; Inactive date
          (,(format "\\(\\[%s\\]\\)" date-re) .
           ((lambda (tag) (svg-tag-make tag :beg 1 :end -1 :margin 0 :face 'org-date))))
          (,(format "\\(\\[%s \\)%s\\]" date-re day-time-re) .
           ((lambda (tag) (svg-tag-make tag :beg 1 :inverse nil :crop-right t :margin 0 :face 'org-date))))
          (,(format "\\[%s \\(%s\\]\\)" date-re day-time-re) .

(defun my/daily-setup ()

***** Org-capture global capture key

Two capture templates: fleeting notes go into today's daily page, quests go under =* Dailies= in =tasks.org=. The global Niri shortcut (=Mod+C=) calls the fleeting template directly, skipping the menu.

This first blocks sets up

#+begin_src emacs-lisp

(defconst +my/org-capture-frame-name "Org-capture to daily note")
(defun +my/org-capture-frame (template) (interactive)
       (if (not (equal +my/org-capture-frame-name (frame-parameter nil 'name)))
           (make-frame `((name . ,+my/org-capture-frame-name))))

       (select-frame-by-name +my/org-capture-frame-name)
       (delete-other-windows)

       (defun org-capture-after-finalize-cleanup ()
         (advice-remove 'org-capture-place-template 'delete-other-windows)
         (remove-hook 'org-capture-after-finalize-hook 'org-capture-after-finalize-cleanup)

         (select-frame-by-name +my/org-capture-frame-name)
         (delete-frame nil))

       (add-hook 'org-capture-after-finalize-hook 'org-capture-after-finalize-cleanup)
       (advice-add #'org-capture-place-template :after 'delete-other-windows)

       (org-capture nil template))
